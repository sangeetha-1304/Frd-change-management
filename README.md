# Frd-change-management

## 📌 Project Overview

### This project contains the Functional Requirement Document (FRD) for a Change Management System designed for the pharmaceutical industry. The system ensures proper tracking, approval, and implementation of changes in compliance with regulatory standards.

## 🎯 Objective

### To provide a structured system for managing changes related to processes, documents, and equipment while ensuring quality and compliance.

## ⚙️ Key Features

### - Change request creation and management
### - Multi-level review and approval workflow
### - Audit trail for tracking all activities
### - Document upload with version control
### - Dashboard for monitoring request status
### - Search and filter functionality

## 🔄 Workflow

### 1. Create Change Request
### 2. Review Request
### 3. Approve or Reject
### 4. Implement Change
### 5. Close Request

## 🧑‍💼 User Roles

### - Initiator
### - Reviewer
### - Approver
### - QA Team
### - Admin

## 📊 Sprint Plan

### - Sprint 1: Requirement analysis and design
### - Sprint 2: Change request module
### - Sprint 3: Approval workflow and notifications
### - Sprint 4: Dashboard, testing, and deployment
